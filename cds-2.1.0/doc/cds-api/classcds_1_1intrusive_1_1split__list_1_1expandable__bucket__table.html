Detailed Description

template<typename GC, typename Node, typename... Options>
class cds::intrusive::split_list::expandable_bucket_table< GC, Node, Options >

Expandable bucket table.

This bucket table can dynamically grow its capacity when necessary up to maximum bucket count.

Template parameter:

  • GC - garbage collector
  • Node - node type, must be derived from split_list::node
  • Options... - options

Options are:

  • opt::allocator - allocator used to allocate bucket table. Default is CDS_DEFAULT_ALLOCATOR
  • opt::memory_model - memory model used. Possible types are opt::v::sequential_consistent, opt::v::relaxed_ordering
