Detailed Description

Thread GC.

Thread's hazard pointer manager.

To use Dynamic Hazard Pointer reclamation schema each thread object must be linked with the object of ThreadGC class that interacts with GarbageCollector global object. The linkage is performed by calling cds::threading::Manager::attachThread() on the start of each thread that uses DHP GC. Before terminating the thread linked to DHP GC it is necessary to call cds::threading::Manager::detachThread().

The ThreadGC object maintains two list:

  • Thread guard list: the list of thread-local guards (linked by pThreadNext field)
  • Free guard list: the list of thread-local free guards (linked by pNextFree field) Free guard list is a subset of thread guard list.
