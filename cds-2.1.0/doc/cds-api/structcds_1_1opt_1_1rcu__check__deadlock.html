template<typename Type>
struct cds::opt::rcu_check_deadlock< Type >

[type-option] RCU check deadlock option setter

The RCU containers can check if a deadlock between read-side critical section and synchronize call is possible. This option specifies a policy for checking this situation. Possible Type is:

  • opt::v::rcu_no_check_deadlock - no deadlock checking
  • opt::v::rcu_assert_deadlock - call assert in debug mode only
  • opt::v::rcu_throw_deadlock - throw an cds::urcu::rcu_deadlock exception when a deadlock is encountered

Usually, the default Type for this option is opt::v::rcu_throw_deadlock.
